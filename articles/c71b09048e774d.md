## イントロダクション

テキストベースのコーディングと違い、Grasshopperではマウスでの作業が多くなってしまう。作業効率を上げるために自然と多用するようになったショートカットや、隠れたマウス操作を書き出してみた。なお、個人的な使用環境がWindowsのみで基本的にはMacに対応していないのでご了解いただきたい。

## ワイヤに関するTips

### 距離のあるコンポーネントに接続

これはスクリーンの外にあるコンポーネントがターゲットとなる。便利なプラグイン(Telepathy等)は存在するが、可能な限り個人の作ったプラグインを利用しない方針。私のやり方は以下のような手順だ。一見複雑で効率が悪そうだが慣れれば自然にできるようになるので試してほしい。

1. 端子(Output)をダブルクリックしてRelayを作る。
2. Relayをカット(`Ctrl` + `C`)。
3. 目的のコンポーネントまで移動。
4. `Ctrl` + `Shift` + `V` で先ほどのRelayをキャンバス中央にペースト。
5. Relayとコンポーネントを接続。(Relayをダブルクリックで消去できる)

**Note:**

- Relayは複数カット可能。
- [移動に関するTips](#移動に関するtips)と組合せで更に効率的。

### 名前の継承

ワイヤは同じ色、同じパターンで、区別がつかない。どんなデータが流れているか一見ではわからない。ワイヤが短ければ解決するが、時として超長くなることがある。従って、ワイヤには名前を必ずつけている。
私は、Relayを作ってそこに名前を書き込んでいる。ワイヤをダブルクリックするとRelayが作成される。(たまにうまくワイヤをダブルクリックできない。場所を変えたり、ズームしたりするとうまくいく。)
データ型に合わせてパラムを利用する時もあるが、Relayの利点はワイヤの途中にいくらでも、ダブルクリックでRelayを追加することができるし、次のRelayは何もしなくても名前を継承してくれるところである。

パラムに対しても名前の継承が簡単にできるTipが存在する。
ワイヤを引き出したまま、`Alt`キーをタップするとワイヤが赤くなる。この状態でつなぐと名前が継承される。

## ファイルに関するTips

Grasshopperには複数開いたファイルを管理するサムネイルビューがある。ブラウザなどの一般的なアプリと同じショートカットキーが使える場合がある。

### ファイルの切り替え

開かれたファイルは「最近使用された順に切り替える」仕様になっている。ショートカットキーは `Ctrl` + `Tab`であるが、押し方で以下の２つの動作がある。

【動作1】 **サムネイルビューを開く**

- `Ctrl`キーを押したまま、`Tab`キーをタップするとサムネイルビューが開き、現在開いている全てのファイルのサムネイルが表示される。サムネイルの順番は先ほど説明した通り、使用された順である。

![サムネイルビュー](/images/thumbnail_view.png =250x)
*サムネイルビュー*

**Note:** Grasshopperウィンドウの右上にファイル名が表示されているが、ここをクリックすると縦のサムネイルバーが現れる。

【動作2】 **2つのファイルの切替え**

- 動作1は、前回開いていたファイルを選択した状態になっている。そこで`Tab`キーと一緒に`Ctrl`キーも放すと前のファイルに移動することになる。
つまり、この動作を連続して行うと2つのファイルのスイッチングを繰り返すことができる。2つのファイルを行き来するのにとても便利だ。

### バックアップ

崇高なプログラマーたちは、バージョニングにGitを使っているのであろう。GrasshopperにはSave Backup (`Ctrl` + `Alt` + `S`) という変わった保存方法が用意されている。
私は何か大きな機能変更を行うとき素早くこのコマンドを実行している。ファイル名にタイムスタンプが追加され元のファイルと区別される。

![save_backup](/images/save_backup.png =250x)
*バックアップ ファイル名*

この時忘れずにRevisions (**File -> Document Properties**) にメモをつける。忘れずにというのはよく忘れるからだ。忘れてもよい。バックアップが残っていればあまり重要なことではない。

![document_properties](/images/document_properties.gif =250x)
*Revision メモ*

**Note:**

- たまったバックアップ ファイルは時々クリーンアップする。VS Codeのマネをして **.history** フォルダに移動するしている。

- もしGitを利用するなら、ファイルの拡張子を **.gh** -> **.ghx** にすれば差分を見ることもできるだろうが時間の無駄だ。
差分を見たければ、画像処理でどこが変更されたか一瞥で分かるよい方法がある。前ファイルのキャンバスからコピーして現在ファイルにペーストすればよい。(この時に即座にグループ化しておくことをおすすめする。) 馬鹿にしてはいけない、人の目はわずかな違いでも見つけてくれるのだ。

- 意外に知らない人が多いが、失ったものを取り戻す方法がある。消してしまってから少しだけ作業が進んでしまった場合、`Ctrl` + `Z` 連打でもどればよい。消す前までもどって必要なものを`Ctrl` + `C` コピー、`Ctrl` + `Y`をホールドして現在までもどって`Ctrl` + `Shift` + `V`で画面の真ん中にペーストするだけだ。

## 移動に関するTips

キャンバス上の移動は、マウス腱鞘炎になるぐらいつらい。私は親指トラックボールを使用しているので親指が痛い。そこで、指が少し動けばポインタが大きく動くように設定している。(細かいコントロールは経験値を上げれば可能だ。)
以下に、助けとなるショートカットを挙げる。

### 全体の表示

[距離のあるコンポーネントに接続](#距離のあるコンポーネントに接続)でも紹介したTipと一緒によく使うショートカット。
遠くのコンポーネントへ出発するときはまずこのショートカットを使う。

- `Ctrl` + `Shift` + `Z`

### 選択の表示

**Zoom Selected**のことだが、デフォルトではショートカットが設定されていないので、私は`Ctrl` + `Shift` + `X`を割り当てている。
使いタイミングは、[全体の表示](#全体の表示)した後、この辺だろうと見当をつけて囲み選択し、このショートカットで拡大している。

キャンバス内の移動は、大抵この2つのショートカットを活用することでマウスのズームやパンを減らしている。

![選択の表示](/images/zoom_selected.png =280x)
*【選択の表示】ショートカットの割り当て*

### 場所を登録する

Grasshopperは標準でキャンバスビュー管理機能がある。キャンバス上好きなところを登録することでクリック一つでいつでもその場所に移動することが出来る。
私の場合、処理ブロックごとに大雑把に登録しておくことで移動を容易にしている。
登録は簡単で、ツールバーにある目玉ボタンを押すとエディタが表示され登録できる。移動する時はプルダウンメニューで移動先をクリックするだけだ。

実はとっても良いプラグインを使っているので紹介したい。**SidePanel**である。プラグインは使用しないなどといったがこれは便利だ。左隅に常駐しているが移動の苦労を考えれば、邪魔ではない。閉じることもできる。
ビューに名前をつけて登録する機能以外あまり使っていないが、このプラグインは多機能で、配置されている個々のコンポーネントやGroupをリスト化してくれ、もちろんクリックすればその場所に連れて行ってくれる。但し検索機能ではなく単にリストアップしてくれるだけだ。コンポーネントやGroupに名前をつける利用者には便利だろう。
ついでに、あまり使っていないが利用価値のある機能がStatesである。Grasshopperの基本機能でもあるが、これはBoolean Toggleの状態やスライダの値をいっぺんに登録できる機能だ。複数の固定パラメータを登録(プリセット)しておいて素早く切り替えるのに便利だ。

[![SidePanel](/images/sidepanel.png =250x)*SidePanel*](<https://www.food4rhino.com/en/app/sidepanel>)

## 可読性に関するTips

どんなに時間が経った後でもGH定義(つまり、Grasshopperのプログラム)の全体を手っ取り早く把握できるように可読性を向上させたい。とはいえ、そのために時間をかけたくない。いかにするかは職場のルールよって異なると思うが、参考までにどのようにしているかを以下に挙げる。

### 縦にセパレートする

`Alt`キーを押しながら、キャンバス上を左マウスボタンでドラッグをすると縦に真っ二つに分けるように空白地帯を作ることはご存じだろう。
しかし、縦に空白地帯は作れないのであろうかと思った方もいるのではないだろうか。
ちゃんと用意されているのである。実は、もう一回`Alt`キーをタップすれば今度は横に真っ二つに分けて縦にも空白地帯を作ることができるのだ。
`Alt`キーは、ドラッグを始めれば放しても大丈夫なことに気がつくだろう。タップをするたびに、横と縦が入れ替わる。

![separate_canvas](/images/separate_canvas.gif =250x)

### グループ化する

どの程度まとまればグループにするのかの基準は、端的には「処理ごとにする」。つまりある処理が完了して次の処理をするときに思い切ってグループ化する。グループ化は、処理全体を選択し`Ctrl` + `G`で行う。
クラスターが便利と聞くが、クラスターに閉じ込めてしまうことによる欠点が顕著で全く良さを感じなかったということがあり利用していない。

### グループの色を変える

グループの色を変えると可読性が向上し、これによって得られる効果は大きい。ちょっと手間だが、色は変えるべきだ。
毎回同じ色を使うべく微調整するつもりはないので、カラーピッカーのコーナーやスライダーの'はじっこ'を目掛けておもいきりドラッグして得られる色しか使わない。
例えば、デバッグのためにデータの可視化をする処理グループは「赤」、レンダリングの処理グループは「黄」、説明のためのScribbleやSketchは「白」でグループ化というようにしている。ほとんどのグループは通常の処理グループであり、通常の処理グループは「デフォルトの色」にしているので煩わしさはほとんどない。

### スケッチ機能の隠れたTip

これは恐らく、あなたのようなヘビー利用者でも知らないのではないかと思うので紹介してみよう。
ツールバーにペンマークのアイコンのSketchがあるのはご存じだろう。特に機能の少ないスケッチだが、キャンバスに描いたものを右クリックしコンテキストメニューを開くとほんとうに少しだけ補足機能がある。
ここに**Send to Rhino (送信)** と **Load from Rhino (置換)** というスケッチの見た目をきちんと整える機能があるのだ。
手順は以下のような順に従う。

1. 以下のようなスケッチをRhinoで描いて美しくしてみよう。
![sketch_1](/images/sketch_1.png =250x)
2. **送信**するとRhino上にスケッチが現れる。その上から線や曲線でなぞるように作図する。
![sketch_2](/images/sketch_2.png =250x)
3. 今回、4色に色分けしたいので、1の元のスケッチをコピー/ペーストを3回繰り返して、4つのコピーを用意する。なぜかずれてしまうが気にしないでほしい。
![sketch_3](/images/sketch_3.gif =330x)
4. 先ほどの4つのコピーを一つずつ**置換**する。下の画像は、Grasshopperに戻って色付け完成図。4つのパーツがずれることなく入れ替わった。最後にグループ化しておくとバラバラになりづらくなる。
![sketch_4](/images/sketch_4.png =250x)

**Note:**

- 細かい話だが、上の完成図をよく見ると線と線が重なってしまって美しくない。そう感じたら、前に持っていきたいパーツを選択し、`Ctrl`+`F` で**最前面に送る**ことが出来る。これは、コンポーネントなどの他のオブジェクトでも同様なので必要なら重なりの順番を変えることができる。(通常は作った順番となっている)

![edit_arrange](/images/edit_arrange.png =450x)
*曲線(黒)を最前面に送る図*

## コピー・ペーストに関するTips

コンポーネントをドラッグして、`Alt`キーを押すと簡単にコピーされることはご存じだと思う。
他には、有益な方法がないか検討した。

### ワイヤを外してくれるコピペ機能

一部を抽出するためにコピペすると、ワイヤがつながった状態でコピーが出来てしまう。それをわざわざはずして使った経験がないだろうか。
そこで、便利なプラグインがあり、これは `Alt` + `D` で不要なワイヤを外してコピーを複製してくれる。~~`Shift` + `Alt` + `D` で、グループを選択すればグループごとコピーを作れる。~~
但し、惜しいことに、パラムのワイヤは外してくれないことを容赦しなければならない。

@[card](https://www.food4rhino.com/en/app/ocd-tools)

### コピーを中央にペースト(配置)する

`Ctrl` + `Shift` + `V` (Paste In Center)

[距離のあるコンポーネントに接続](#距離のあるコンポーネントに接続)でも紹介したが、通常のコピペはコピー元のすぐそばに重ねたようにペーストされるが、これはキャンバスの中央にペーストする機能。どこに移動したとしても、とにかく今見ているキャンバス上の中央にペーストしてくれる。

## デバッグに関するTips

### データフローをモニタする

`Ctrl` + `Shift` + `D` でData Viewerがぽんと表示される。もう一回するともう一つ表示される。表示されなければ、いったんキャンバスをクリックしてから行おう。おそらく表示できるData Viewerに制限はない。
どのようなデータが流れているか検討しなければならない時はPanelを利用すると思うが、Data Viewerのほうが圧倒的に優れている。以下に利点を挙げる。

- スクロールがやりやすい。スクロールバー ハンドルが大きくてつかみやすい。マウスのホイールに反応してくれる。しかも、ホイールクリックのオートスクロールもできる。素晴らしいです。
- ブランチがいくつで、アイテムがそれぞれいくつというデータツリー構造が一見してわかる。
- キャンバスに依存せず、ウインドウを移動したり大きさを変えてもスクリーンのポジションは変化しない。
- リアルタイムにデータをモニタリングできる。オプションとして**Auto Update (クリックした端子)** と、**Assign Target (特定の端子)** が選択できるが、実際大きな違いはない。私はAuto Updateしか使わない。
