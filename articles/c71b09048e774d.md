## イントロダクション

テキストベースのコーディングと違い、Grasshopperではマウスでの作業が多くなってしまう。作業効率を上げるために自然と多用するようなになったショートカットや、隠れたマウス操作を書き出してみた。なお、個人的な使用環境がWindowsのみで基本的にはMacに対応していないのでご了解いただきたい。

## ワイヤに関するTips

### 距離のあるコンポーネントに接続する

これはスクリーンの外にあるコンポーネントがターゲットとなる。便利なプラグイン(Telepathy等)は存在するが、可能な限り個人の作ったプラグインを利用しない方針。私のやり方は以下のような手順だ。一見複雑で効率が悪そうだが慣れれば自然にできるようになるので試してほしい。

1. 端子(Output)をダブルクリックしてRelayを作る。
2. Relayをカット(`Ctrl` + `C`)。
3. 目的のコンポーネントまで移動。
4. `Ctrl` + `Shift` + `V` で先ほどのRelayをキャンバス中央にペースト。
5. Relayとコンポーネントを接続。(Relayをダブルクリックで消去できる)

Note:
    - ワイヤは複数接続可能。
    - [移動に関するTips](#移動に関するtips)と組合せで更に効率的。

### 名前の継承

ワイヤは同じ色、同じパターンで、区別がつかない。どんなデータが流れているか一見ではわからない。ワイヤが短ければ解決するが、時として超長くなることがある。従って、ワイヤには名前を必ずつけている。
私は、リレーを作ってそこに名前を書き込んでいる。ワイヤをダブルクリックするとリレーが作成される。(たまにうまくワイヤをダブルクリックできない。場所を変えたり、ズームしたりするとうまくいく。)
データ型に合わせてパラムを利用する時もあるが、リレーの利点はワイヤの途中にいくらでも、ダブルクリックでリレーを追加することができるし、次のリレーは何もしなくても名前を継承してくれるところである。

パラムに対しても名前の継承が簡単にできるTipが存在する。
ワイヤを引き出したまま、`Alt`キーをタップするとワイヤが赤くなる。この状態でつなぐと名前が継承される。

## ファイルに関するTips

Grasshopperには複数開いたファイルを管理するサムネイルビューがある。ブラウザなどの一般アプリと同じショートカットキーが使える場合がある。

### ファイルの切り替え

開かれたファイルは「最近使用された順に切り替える」仕様になっている。ショートカットキーは `Ctrl` + `Tab`であるが、押し方で以下の２つの動作がある。

【動作1】 **サムネイルビューを開く**

- `Ctrl`キーを押したまま、`Tab`キーをタップするとサムネイルビューが開き、現在開いている全てのファイルのサムネイルが表示される。サムネイルの順番は先ほど説明した通り、使用された順である。

![サムネイルビュー](/images/thumbnail_view.png =250x)
*サムネイルビュー*

Note: Grasshopperウィンドウの右上にファイル名が表示されているが、ここをクリックすると縦のサムネイルバーが現れる。

【動作2】 **2つのファイルの切替え**

- 動作1は、前回開いていたファイルを選択した状態になっている。そこで`Tab`キーと一緒に`Ctrl`キーも放すと前のファイルに移動することになる。
つまり、この動作を連続して行うと2つのファイルのスイッチングを繰り返すことができる。2つのファイルを行き来するのにとても便利だ。

### バックアップ

崇高なプログラマーたちは、バージョ二ングにGitを使っているのであろう。GrasshopperにはSave Backup(`Ctrl` + `Alt` + `S`)という変わった保存方法が用意されている。
私は何か大きな機能変更を行うとき素早くこのコマンドを実行している。ファイル名にタイムスタンプが追加され元のファイルと区別される。

![save_backup](/images/save_backup.png =250x)
*バックアップ ファイル名*

この時忘れずにRevisions (**File -> Document Properties**) にメモをつける。忘れずにというのはよく忘れるからだ。忘れてもよい。あまり重要なことではない。

![document_properties](/images/document_properties.gif =250x)
*Revision メモ*

Note:

- たまったバックアップ ファイルは時々クリーンアップする。VS Codeのマネをして **.history** フォルダに移動するのがよい。

- もしGitを利用するなら、ファイルの拡張子を **.gh** -> **.ghx** にすれば差分を見ることもできるだろうが時間の無駄だ。
差分を見たければ、画像処理でどこが変更されたか一瞥で分かるよい方法がある。前ファイルのキャンバスからコピーして現在ファイルにペーストすればよい。(この時に即座にグループ化しておくことをおすすめする。) 馬鹿にしてはいけない、人の目はわずかな違いでも見つけてくれるのだ。

- 以外に知らない人が多いが、失ったものを取り戻す方法がある。消してしまってから少しだけ作業が進んでしまった場合、`Ctrl` + `Z` 連打でもどればよい。消す前までもどって必要なものを`Ctrl` + `C` コピー、`Ctrl` + `Y`をホールドして現在までもどって`Ctrl` + `Shift` + `V`で画面の真ん中にペーストするだけだ。

## 移動に関するTips

キャンパス上の移動は、マウス腱鞘炎になるぐらいつらい。私は親指トラックボールを使用しているので親指が痛い。そこで、指が少し動けばポインタが大きく動くように設定している。(細かいコントロールは経験値を上げれば可能だ。)
以下に、助けとなるショートカットを挙げる。

### 全体を表示

[距離のあるコンポーネントに接続する](#距離のあるコンポーネントに接続する)でも紹介したTipと一緒によく使うショートカット。
遠くのコンポーネントへ出発するときはまずこのショートカットを使う。

- `Ctrl` + `Shift` + `Z`

### 選択を表示

**Zoom Selected**のことだが、デフォルトではショートカットが設定されていないので、私は`Ctrl` + `Shift` + `X`を割当てている。
[全体を表示](#全体を表示)した後、この辺だろうと見当をつけて囲んで選択、このショートカットで拡大される。

キャンパス内の移動は、大抵この2つのショートカットを活用することでマウスのズームやパンを減らしている。

![選択を表示](/images/zoom_selected.png =250x)
*【選択を表示】ショートカット*

### 場所を登録

Grasshopperは標準でキャンバスビュー管理機能がある。キャンバス上好きなところを登録することでクリック一つでいつでもその場所に移動することが出来る。
私の場合、処理ブロックごとに大雑把に登録しておくことで移動を容易にしている。
登録は簡単で、Canvas Toolbarの目玉ボタンを押すとEditorが表示され登録できる。移動するときはプルダウンメニューで移動先をクリックするだけだ。

実はとっても良いプラグインを使っているので紹介したい。**SidePanel**である。プラグインは使用しないなどといったがこれは便利だ。左隅に常駐しているが移動の苦労を考えれば、邪魔ではない。
ビューに名前をつけて登録する機能以外あまり使っていないが、このプラグインは多機能で、すでに配置されている個々のコンポーネントやGroupをリスト化してくれ、もちろんクリックすればその場所に連れて行ってくれる。但し検索機能ではなく単にリストアップしてくれるだけだ。コンポーネントやGroupに名前をつけるユーザーには便利だろう。
ついでに、あまり使っていないが利用価値のある機能がStatesである。これはBoolean Toggleの状態やスライダの値をいっぺんに登録できる機能だ。複数の固定パラメータを登録しておいて素早く切り替えるのに便利だ。

*SidePanel*

## 可読性に関するTips

どんなに時間が経ったGH定義(つまり、Grasshopperのプログラム)でも、全体を手っ取り早く把握するには可読性を向上させなければならない。とはいえ、そのために時間をかけたくない。いかにするかは、職場のルールよって異なると思うがどのようにしているかを以下に挙げる。

### グループ化する

どの程度まとまればグループにするのかの基準は、端的には「処理ごとにする」、つまりある処理が完了して次の処理をするときに思い切ってグループ化する。グループ化は、処理全体を選択し'Ctrl' + 'G'で行う。
クラスターが便利と聞くが、クラスターに閉じ込めてしまうことによる欠点が顕著であまり良さ感じなかったということがあり利用していない。

### グループの色を変える

すばやくグループの色を変えるためのTipはないが、この可読性向上による得られる効果は大きい。ちょっと手間だが、色は変えるべきだ。
とはいえ、毎回同じ色を使うべく微調整するつもりはなく、カラーピッカーのコーナーやスライダーの'はじっこ'目掛けておもいきりドラッグして得られる色しか使わない。
例えば、デバッグのためにデータの可視化をする処理グループは「赤」、レンダリングの処理グループは「黄」、説明のためのScribbleやSketchは「白」でグループ化というようにしている。ほとんどのグループは通常の処理グループであり、通常の処理グループは「デフォルトの色」にしているので煩わしさはない。

### スケッチ機能の隠れたTip

これは恐らく、あなたのようなヘビーユーザーでも知らないのではないかと思うので紹介してみよう。
ツールバーにペンマークのアイコンのSketchがあるのはご存じだろう。特に機能の少ないスケッチだが、キャンバスに描いたものを右クリックしコンテキストメニューを開くとほんとうに少しだけだが機能の補足がある。
ここに**Send to Rhino (送信)** と **Load from Rhino (取替)** というスケッチの見た目をきちんと整える機能があるのだ。
手順は以下のような順に従う。

1. 以下のようなスケッチをRhinoで描いた線画と取り換えてみよう。
![sketch_1](/images/sketch_1.png =250x)
2. **送信**するとRhino上にスケッチが現れる。その上から線や曲線でなぞるように作図する。
![sketch_2](/images/sketch_2.png =250x)
3. 今回、4色に色分けしたいので、1の元のスケッチをコピー/ペーストを3回繰り返して、4つのコピーを用意する。なぜかずれてしまうが気にしないでほしい。
![sketch_1](/images/sketch_3.gif =250x)
4. 先ほどの4つのコピーを一つずつ**取替**る。下の画像は、Grasshopperに戻って色付け完成図。4つのパーツがずれることなく入れ替わった。最後にグループ化しておくとバラバラになりずらくなる。
![sketch_4](/images/sketch_4.png =250x)
